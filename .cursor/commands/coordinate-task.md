# Coordinate Task (CoordinatorAgent)

Use as a checklist when orchestrating subagents.

1. **Decompose** the user goal into **N** parallelizable tasks.
2. **Efficiency gate (when applicable):** If **N > 2**, or the work is **web / many URLs / doc fetch / MCP-heavy**, or **broad research** without scope, run **`efficiency-inspector-agent`** first with **`planned_tasks[]`** and merge JSON. Obey **`halt_or_narrow`** when **priority** is **high**. See [`.cursor/rules/coordinator-agent.mdc`](../rules/coordinator-agent.mdc) step 4.
3. For each task, spawn **Task** with **`subagent_type` = YAML `name`** from the chosen [`.cursor/agents/`](../agents/) file (e.g. `repo-explorer-agent`, `api-integration-agent`). Persona files: `repo-explorer-agent.md`, etc. The **`name`** field in frontmatter is what the Task tool must receive; slash commands are optional UX. **Avoid** defaulting to **`generalPurpose`**â€”if you use it, add **one line** in the task prompt explaining why no specialist fit.
4. **Routing:** Full table in [`.cursor/rules/coordinator-agent.mdc`](../rules/coordinator-agent.mdc). Short map: repo map / conventions â†’ `repo-explorer-agent`; quick codebase search â†’ `explore`; DB â†’ `database-explorer-agent`; API contracts â†’ `api-integration-agent`; SDK samples â†’ `sdk-example-agent`; **new subagent file** â†’ `subagent-author-agent`; **minimal web research** â†’ `reference-synthesis-agent`; **doc snippets / clipped sections** â†’ `doc-snippet-agent`; changelogs â†’ `release-notes-agent`; env matrix â†’ `compatibility-matrix-agent`; secrets/CORS â†’ `secret-config-agent`; a11y â†’ `accessibility-agent`; diff vs contract â†’ `bughunting-agent`; **CI log triage** â†’ `ci-failure-agent`; **tests** â†’ `test-author-agent`; **authorized security review** â†’ `app-security-review-agent`; pre-flight efficiency â†’ `efficiency-inspector-agent`; **refactor** repeated commands â†’ `script-optimizer-agent`; **greenfield** small tools â†’ `tool-builder-agent`; verified findings â†’ `cataloger-agent`.
5. Per task, include: input JSON (`task_id`, scope, constraints); paths `.cursor/scratchpad.md`, `.cursor/VerifiedFindings.md`, `.cursor/UNEXPECTEDRESULTS.md` (if incident logging applies), `.cursor/repo-map.md`; optionally **`.cursor/capabilities-inventory.md`** for onboarding or heavy orchestration only; expected output JSON envelope `schema_version: "1"` (optional root-level **`coordinator_alerts[]`** per **`@unexpected-coordinator-alert`**). For parse-heavy work, use **`@deterministic-parse-first`** or **`script-optimizer-agent`** / **`tool-builder-agent`**.
6. **Validate** responses: `node scripts/validate-subagent-output.mjs <file.json>`
7. **Merge** only valid JSON. Treat `empty_result` as success-with-information.
8. **Promote** to VerifiedFindings **only** via CatalogerAgent (or switch to Cataloger persona yourself with the same rules). **Log unexpected incidents** to **`.cursor/UNEXPECTEDRESULTS.md`** **only** via CatalogerAgent when merging **`coordinator_alerts[]`** or when asked (skills **`@unexpected-coordinator-alert`**, **`@unexpected-results-catalog`**).

9. **OWASP LLM (2025):** Specialists use **in-persona bullets** by default; they attach **`@owasp-llm-2025-baseline`** when **unsure** or inputs are **surprising**. **`app-security-review-agent`** and **`subagent-author-agent`** always load the baseline. Coordinator may attach the baseline on **high-risk** runs. Validate merged JSON before treating outputs as facts.

**MCP**: Ensure `.cursor/allow-mcp` or subagent session before expecting MCP tools to work (see hooks). High-risk tools (e.g. binary analysis) also need `.cursor/allow-pentest-mcp` plus subagent sessionâ€”see [`.cursor/rules/pentest-mcp.mdc`](../rules/pentest-mcp.mdc).
